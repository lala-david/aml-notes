# Submissions — 감독/기관 공식 검수 요청

> KoFIU·FSC·FSS·DAXA·로펌 등에 공식 검수·협력을 요청하는 **바로 보낼 수 있는 이메일 드래프트**.
> 마지막 갱신: 2026-04-21

---

## 사용 순서

1. **타겟 선정** — Tier 1(KoFIU) 먼저. 2~4주 내 응답 없으면 Tier 2(DAXA) 병행.
2. **드래프트 선택** — `01-XX-ko.md` 열어 `[수신자명]`·`[이름]`·`[연락처]`·`[소속]` 변수 채우기
3. **1차 자료 리뷰 첨부** — 이메일에 첨부할 핵심 파일 경로 확인 (각 드래프트에 명시됨)
4. **발송** — 공식 메일 (국·공립기관) 또는 LinkedIn·이메일 (민간)
5. **응답 로그** — 이 폴더에 `00-response-log.md` 파일을 만들어 응답·회의록 기록

## Tips

- **첫 이메일은 짧게** (5~7줄) — 상세는 첨부 PDF 또는 GitHub 링크로
- **Ask는 명확히** — "10분 회의"·"이메일 회신만"·"검토 후 의견 회신" 등 reachable goal 1개
- **예의·격식** — 공공기관에 보낼 때는 반드시 공식 양식·존칭 사용
- **GitHub 링크 공개** — 상대가 즉시 확인 가능하도록
- **한국어 우선** — 국내 기관은 한국어, 국제 관계자는 06-en 버전 활용
- **1인 1기관 담당** — 응답 추적 혼선 방지

## 발송 후 체크리스트

- [ ] 24시간 내 수신 확인 (자동 회신/미수신 등)
- [ ] 1주 차 reminder 메일 (공손하게)
- [ ] 2주 차 다른 경로 병행 (LinkedIn 등)
- [ ] 응답 수신 시 `00-response-log.md` 기록
- [ ] 검수 의견은 `kofiu-review`·`fsc-review` 라벨로 Issue 생성
- [ ] 7일 내 시정 commit → 결과 보고 회신
